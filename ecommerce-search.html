What I Learned Building E-commerce Search

Why keyword search fails, why vector alone isn't enough, and what actually works in production.

Search looks simple on the surface. A user types a query, and the system returns results. But in practice, search is not just a retrieval problem. It is an intent understanding problem. This becomes especially clear in e-commerce, where users rarely express their intent in clean, structured keywords.

Phase 1: Lexical Search (Matching What Users Say)

Most early e-commerce search systems rely on lexical matching techniques like BM25. They treat queries and products as collections of tokens and rank results based on keyword overlap.

This approach works well when users and systems "speak the same language."

It's fast, scalable, and deterministic. So it's easy to debug and reason about why specific results come out.

That being said, the limitation is that it retrieves what users type or say, not what they "mean".

I saw this clearly while working on Samsung's on-site search. For queries like "galaxy earphones", the system often returned Galaxy phones at the top of the results. From a keyword perspective, this was correct. From a user perspective, it was clearly wrong.

This gap between keyword matching and user intent is where most search problems begin.

Phase 2: The Patchwork Era (Trying to Fix It Manually)

To address these gaps, teams typically add layers of heuristics, such as synonym dictionaries, query rewriting, typo correction, redirect rules, and manual ranking adjustments.

These fixes help in specific cases, but they don't scale. Rules start conflicting with each other, long-tail queries remain unsolved, and maintenance cost grows quickly.

The system doesn't actually get smarter. It just becomes more complicated.

Phase 3: The Shift to Semantic Search

Semantic search changes the approach by representing queries and products as embeddings. Instead of matching keywords, the system matches meaning.

This enables better handling of synonyms, improved long-tail query coverage, and more flexible matching between queries and products. At first glance, this seems like the solution.

But in practice, it introduces a different set of challenges. Results become less predictable, relevance is harder to explain, evaluation becomes more complex, and infrastructure cost increases.

Semantic search improves understanding, but reduces manual control. The heuristic layers built in Phase 2 become less necessary, but so does the ability to directly intervene when something goes wrong.

The Reality: Hybrid Systems

In production, no system is purely lexical or purely semantic. Modern search systems are hybrid.

Lexical search provides precision and stability. Semantic search provides recall and intent understanding. The challenge is how to combine them effectively.

This became clear in my own work. I initially assumed the issue with accessory queries was a UI problem and proposed separating them into a dedicated tab. However, behavioral data showed that users rarely interacted with tabs, regardless of query type.

This shifted the focus away from UI and toward improving retrieval and ranking.

I analyzed queries with significantly low CTR and found that many of them were intent-mismatch cases. This led us to expand candidate retrieval beyond keyword matching and incorporate semantic signals for better intent coverage.

To measure progress, I tracked metrics at two levels. On the model side: MRR, precision@k, and recall to evaluate retrieval quality. On the user side: 1st-result CTR and overall search result CTR to validate whether improvements actually changed behavior.

As a result, we observed a meaningful lift in CTR for affected queries and a reduction in query reformulation rates.

Ultimately, search quality wasn't determined by any single model, but by how lexical signals, semantic signals, and behavioral data worked together.

Final Insight

Search is not just about retrieving results. It is a system that must align user intent with product representation, while balancing relevance, business goals, and user trust.

Moving from keyword search to vector search is not a linear upgrade. It is a shift in how we think about the problem.

Hybrid search is not a transition phase. It is the system.

There is no single "best" search system, only better trade-offs.
